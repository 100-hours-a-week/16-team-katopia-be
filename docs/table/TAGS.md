# Tags

## 1) 테이블 설명
태그 마스터 테이블.

## 2) 관계
- tags(1) : post_tags(N)

## 3) 설계 근거
- 태그명 변경/정규화 정책을 고려해 대리키 기반으로 관리한다.
- 중복 생성을 방지하기 위해 name 유니크 제약을 둔다.

## 4) 컬럼 정의
No | 컬럼 | 타입 | 제약 | 설명
---|---|---|---|---
1 | id | BIGINT | pk, nn | 태그 ID
2 | name | VARCHAR(20) | unique, nn | 태그명

## 5) 인덱스/제약
- `uk_tags_name (name)`
- `idx_tags_name (name)`

## 6) 운영 정책
- 태그명은 정규화(앞의 `#` 제거) 후 저장
- 중복 생성 방지 실패 시 유니크 제약으로 차단
