# Member Styles

## 1. 테이블 설명
회원의 선호 스타일을 저장하는 테이블(ElementCollection).
스타일은 고정된 Enum 값을 사용하며, 서비스 정책상 최대 2개까지 허용한다.

### 연관 관계
- members(1) : member_styles(N)

## 2. 기능적 역할
- 사용자 프로필의 일부이므로 별도 엔티티가 아닌 ElementCollection으로 관리한다.
- style 중복은 서비스 검증과 유니크 제약으로 방지한다.

## 3. 컬럼 정의
No | 컬럼 | 타입 | 제약 | 설명
---|---|---|---|---
1 | member_id | BIGINT | nn | 회원 ID
2 | style | VARCHAR(30) | nn | 스타일 값(ENUM 문자열)

## 4. 인덱스/제약
| 인덱스/제약 | 근거 |
| --- | --- |
| 없음 | 현재 구조는 `member_id` 기준 단순 종속 컬렉션이며, 중복/최대 개수는 애플리케이션 검증으로 처리하고 있어 별도 인덱스를 두지 않는다 |

## 5. 운영 정책
- 스타일 목록은 검증기로 유효성/최대 개수 제한을 강제한다.
- 저장 시 대문자 Enum 문자열로 정규화한다.
- 스타일 추천 로직 고도화 시 별도 엔티티 관리를 고려한다.
